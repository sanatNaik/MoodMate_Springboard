🎵 MoodMate
Emotion-Aware Music Recommendation System
________________


Overview
MoodMate is an AI-powered system that detects a user's emotional state from text input and recommends music aligned with that emotion.
The project integrates:
* Transformer-based NLP (DistilBERT)
* Deep learning classification
* Content-based music recommendation
________________


Features
* Emotion detection using a fine-tuned DistilBERT model
* Handles class imbalance using weighted loss
* Evaluation using accuracy, precision, recall, and F1-score
* Music recommendation using TF-IDF + cosine similarity
* Custom frontend (HTML/CSS/JS) with Streamlit backend
________________


Model Architecture
* Base Model: distilbert-base-uncased
* Custom Classification Head:
   * Linear (768 → 256)
   * ReLU + Dropout
   * Linear (256 → 128)
   * ReLU + Dropout
   * Linear (128 → 5)
* Loss Function:
   * CrossEntropyLoss with class weights
________________


6. Datasets
6.1 Emotion Dataset
The emotion detection model was trained on a text-based dataset containing sentences labeled with emotional categories.
* Source: Hugging Face (boltuix/emotions-dataset)
* Type: Supervised text classification dataset
* Data Format:
   * Input: Sentence / text
   * Output: Emotion label
* Original Classes:
 Multiple fine-grained emotions such as happiness, sadness, anger, fear, surprise, etc.
________________


Preprocessing
   * Tokenization performed using a pretrained DistilBERT tokenizer
   * The tokenizer is uncased, handling text normalization automatically
   * Labels were mapped to reduced emotion categories
________________


Final Emotion Classes Used
   * Happy
   * Sad
   * Calm
   * Energetic
   * Angry
________________


Reason for Reduction
   * Simplifies the classification task
   * Improves model generalization
   * Aligns better with music recommendation categories
________________


Class Imbalance Handling
   * Class weights were computed using:
compute_class_weight()
   * Used in loss function:
CrossEntropyLoss(weight=class_weights)
This ensures balanced learning across all emotion classes.
________________


Data Splitting
   * 70% Training
   * 15% Validation
   * 15% Testing
   * Stratified splitting used to maintain class distribution
________________


6.2 Music Dataset
The music dataset is used to recommend songs based on the detected emotion.
   * Type: Song metadata dataset
   * Scale: Thousands of songs with ~35,000 unique tag combinations
________________


Attributes
   * Song name
   * Artist
   * Spotify track ID
   * Preview URL
   * Genre/mood tags
________________


6.3 Data Preprocessing (Music)
The raw music data contained highly diverse and inconsistent tags.
Steps performed:
   * Converted tags to lowercase
   * Replaced underscores with spaces
   * Removed redundant and noisy tokens
   * Combined multiple tags into a single field (clean_tags)
   * Reduced ~35,000 tag combinations into meaningful groups
________________


6.4 Tag Normalization
Tags were mapped into canonical categories:
   * Rock, Metal, Pop, Hip-hop, EDM
   * Classical, Acoustic, Instrumental
   * Ambient, Lo-fi
   * Emotional tags: happy, sad, calm, energetic, angry
________________


6.5 Emotion–Tag Mapping
Each detected emotion is mapped to relevant tags:
   * Happy → pop, happy
   * Sad → acoustic, sad
   * Calm → classical, instrumental, ambient, lo-fi
   * Energetic → rock, hip-hop, edm
   * Angry → metal
________________


6.6 Feature Extraction
   * Technique: TF-IDF (Term Frequency–Inverse Document Frequency)
   * Converts text-based tags into numerical vectors
   * Captures importance of tags across the dataset
________________


6.7 Similarity Computation
   * Method: Cosine Similarity
   * Compares:
   * Emotion query vector
   * Song tag vectors
   * Top-K similar songs are recommended


________________


Training Details
   * Train/Validation/Test Split:
   * 70% Train
   * 15% Validation
   * 15% Test
   * Training Strategy:
   * Stratified splitting
   * Batch size: 32
   * Epochs: 15
   * Learning rate: 1e-5
   * Evaluation Metrics:
   * Accuracy
   * Precision (Macro)
   * Recall (Macro)
   * F1 Score (Macro)
________________


Evaluation
   * Confusion Matrix visualization
   * Classification Report generated
   * Training vs Validation loss plotted
________________
 Recommendation System
   * Uses TF-IDF vectorization on song tags
   * Computes cosine similarity between:
   * Emotion-based query
   * Song metadata

   * Sad → acoustic, sad
   * Calm → classical, ambient, lo-fi
   * Energetic → rock, edm, hip-hop
